# Contributing to TechiesLog

Thank you for your interest in contributing to TechiesLog! We welcome contributions from the community.

## How to Contribute

### Reporting Bugs
- Check if the bug has already been reported in [Issues](https://github.com/vinoth322006/techieslog/issues)
- If not, create a new issue with:
  - Clear title and description
  - Steps to reproduce
  - Expected vs actual behavior
  - Screenshots if applicable
  - Device and Android version

### Suggesting Features
- Check existing [Issues](https://github.com/vinoth322006/techieslog/issues) for similar suggestions
- Create a new issue with:
  - Clear description of the feature
  - Use case and benefits
  - Mockups or examples if applicable

### Code Contributions

#### Setup Development Environment
1. Install Flutter SDK (latest stable version)
2. Clone the repository:
   ```bash
   git clone https://github.com/vinoth322006/techieslog.git
   cd techieslog
   ```
3. Install dependencies:
   ```bash
   flutter pub get
   ```
4. Run the app:
   ```bash
   flutter run
   ```

#### Making Changes
1. Fork the repository
2. Create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes following our coding standards
4. Test your changes thoroughly
5. Commit with clear messages:
   ```bash
   git commit -m "Add: Brief description of changes"
   ```
6. Push to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
7. Create a Pull Request

#### Coding Standards
- Follow Flutter/Dart style guide
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused
- Write tests for new features
- Ensure code passes `flutter analyze`

#### Pull Request Guidelines
- Reference related issues
- Describe what changes were made and why
- Include screenshots for UI changes
- Ensure all tests pass
- Keep PRs focused on a single feature/fix

## Code of Conduct
- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Maintain a harassment-free environment

## Questions?
Feel free to open an issue for any questions or reach out to the maintainers.
